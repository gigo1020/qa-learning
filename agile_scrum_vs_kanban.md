# Scrum vs Kanban Comparison

| Criteria | Scrum | Kanban |
|----------|-------|--------|
| **Iterations** | Fixed-length sprints (1–4 weeks) | Continuous flow (no sprints) |
| **Roles** | Product Owner, Scrum Master, Development Team | No mandatory roles (sometimes a Flow Manager) |
| **Flexibility** | Changes not allowed during sprint; re-prioritization only at next planning | Changes can be made anytime |
| **Metrics** | Velocity (story points per sprint) | Lead Time, Cycle Time, WIP limits |
| **Board reset** | Board is cleared after each sprint | Board is persistent, no reset |
| **Work in Progress (WIP)** | Limited by sprint capacity (implicit) | Explicit WIP limits per column |
| **Best for** | Projects with changing priorities, need for regular feedback | Operations, support, continuous delivery with stable priorities |
