# QA Learning — Tester's Portfolio

**Author:** Daria Koroleva  
**Goal:** Demonstrate skills in manual testing, test design, Git, Jira, and basic QA tools.

## About

This repository was created during a self‑study journey from zero. It contains practical tasks, test cases, checklists, bug reports, and future automation examples.

## Structure

- [`theory_examples.txt`](theory_examples.txt) — equivalence classes, boundary values, test design techniques.
- [`test_cases_login.md`](test_cases_login.md) — 10 test cases for a login form (positive, negative, boundary, XSS, SQL).
- [`checklist_login.md`](checklist_login.md) — 15‑point checklist for login form (UI/UX, security, negatives).
- [`bug-reports/`](bug-reports/) — Jira bug report screenshots (3 bugs with priority/severity).
- [`test_design_techniques.md`](test_design_techniques.md) — flashcards for test design (English).
- [`yandex_test_assignment.md`](yandex_test_assignment.md) — test assignment for Yandex: 10 test cases for search + exploratory testing of the infinite feed (bugs, risks, ambiguities).

## Completed topics

### Block 1. Testing theory
- Levels and types, verification vs validation, error/defect/failure.
- Equivalence partitioning, boundary values, pairwise testing, decision tables, state transition.

### Block 2. Test design & manual testing
- 10 test cases for login form (positive, negative, edge cases, XSS, SQL injection).
- Checklist for login form (15 items).

### Block 3. Git & GitHub
- Branches (`feature/`), commits, pull requests, merging.
- Branch protection rules (`main` branch).

### Block 4. Jira & bug reporting
- Project creation, issue creation (bug), priority & severity.
- Export to PNG, adding to portfolio.

### Block 5. Pet project: Yandex test assignment
- 10 test cases for search redirection (Enter, button, suggestion, empty query, long query, special characters, XSS, SQL).
- Exploratory testing of the infinite feed: bugs, risks, ambiguities (with arguments).
