## API
- A programming interface that allows different programs to interact with each other and transfer data

## REST API
- A programming interface that follows REST principles. Each resource has its own URL. The client sends requests to the server. The server returns responses in JSON or XML format. This defines how to construct the URL and what response codes to return to the client. HTTP methods correspond to CRUD. The REST API does not store session state on the server between requests. Each request is a separate request to the server. The REST API uses caching to reduce server load

## CRUD
- Create, Read, Update, Delete. Corresponds to HTTP requests: POST, GET, PUT/PATCH, and DELETE.

## OpenAPI
- The specification is usually written in JSON or YAML format and consists of sections: info, servers, paths, components/schemas

## Swagger
- A set of tools for working with the specification, including Swagger UI, which generates interactive documentation. Swagger UI – interactive documentation where you can send real requests
